# QA Testing Portfolio

This repository contains my hands-on QA practice projects focused on manual testing, system testing, end-to-end testing, defect reporting, SQL validation, API testing, and beginner-level automation. The main project in this portfolio uses OpenCart, which provides both a storefront demo and an administration demo for realistic e-commerce testing practice.

## About This Repository

This portfolio is built to demonstrate practical QA skills through real project work instead of theory alone. The goal is to show how testing is planned, executed, documented, and reported in a structured way using a real e-commerce application flow. OpenCart is a strong practice platform because it includes important business areas such as products, shipping methods, payment gateways, taxes, and order handling.

## Main Project: OpenCart QA Testing

The core project in this repository is based on OpenCart. OpenCart offers both a frontend store experience and an admin-side management experience, which makes it useful for practicing system testing and end-to-end testing across customer and admin workflows.

### Scope Covered

The OpenCart project focuses on:
- Frontend testing: home page, search, categories, product page, cart, and checkout
- Admin-side validation: order visibility, order details, and related data checks
- End-to-end business flows from product selection to order verification
- Negative testing and validation testing
- Defect reporting and execution tracking
- Supporting QA skills using SQL, API concepts, and Selenium basics

## Skills Demonstrated

This repository is being used to build and demonstrate the following QA skills:

## Folder Overview

- `docs/` – project notes, test plan, and summary reports
- `test-cases/` – detailed test cases and scenario sheets
- `defects/` – bug reports with severity, priority, and reproduction steps
- `rtm/` – requirement traceability matrix
- `execution/` – execution log and test run tracking
- `screenshots/` – evidence collected during testing
- `sql/` – SQL queries and backend validation notes
- `api/` – API testing notes and Postman practice
- `automation/` – basic Selenium framework and scripts
